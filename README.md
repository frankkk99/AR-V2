# ป่าดงพงไพร · WebAR

AR แบบ markerless เปิดผ่านเบราว์เซอร์บนมือถือผู้ชม ไม่ต้องติดตั้งแอป

## วิธี deploy

โครงสร้างเป็น static ทั้งหมด อัปขึ้น Vercel ได้เลย

```
vercel deploy --prod
```

**ต้องเป็น https เท่านั้น** เบราว์เซอร์ไม่ยอมให้ใช้กล้องกับ http
เสร็จแล้วเอา URL ไปทำ QR ติดที่ป้ายนิทรรศการ

## โครงสร้าง

```
index.html              ตัวแอปทั้งหมดอยู่ในไฟล์นี้ไฟล์เดียว
video/mushroom.mp4      11.9 MB รวมสามคลิป
video/tulip.mp4
video/butterfly.mp4
poster/*.png            ภาพในเมนูเลือก
```

## แก้ชื่อและขนาดวัตถุ

เปิด `index.html` หา `const SPECIMENS` ใกล้ ๆ บรรทัด 130 แก้ได้ที่เดียวจบ

| ฟิลด์ | ความหมาย |
|---|---|
| `th` / `sci` / `code` | ชื่อไทย / ชื่อวิทยาศาสตร์ / รหัสตัวอย่าง ขึ้นบนป้ายใน AR |
| `width` | ความกว้างของ**เฟรมวิดีโอ**ในโลกจริง หน่วยเมตร ไม่ใช่ความกว้างตัววัตถุ |
| `baseY` | ความสูงจากพื้นถึงกึ่งกลางเฟรม ใช้ให้ผีเสื้อลอย ส่วนเห็ดกับกระเจียวติดพื้น |

> ชื่อวิทยาศาสตร์ที่ใส่ไว้เป็นค่าตั้งต้น ควรให้ภัณฑารักษ์ยืนยันก่อนเปิดจริง

## QR แยก 3 จุดจัดแสดง

ถ้าไม่ต่อ query string ผู้ชมจะเห็นหน้าเมนูเลือกทั้ง 3 ชนิดตามปกติ
แต่ถ้าอยากทำ QR แยกจุด ให้แต่ละจุดชี้ไปคนละ URL แบบนี้ (แอปจะข้ามหน้าเลือกพื้น/โต๊ะและเมนูไปเลย เพราะทั้ง 3 จุดวางบนพื้นห้อง):

| จุด | สิ่งมีชีวิต | URL ที่เอาไปทำ QR |
|---|---|---|
| 1 | ดอกกระเจียวแดง | `https://<โดเมน>/point-1/` |
| 2 | ผีเสื้อ | `https://<โดเมน>/point-2/` |
| 3 | เห็ดป่าเต็งรัง | `https://<โดเมน>/point-3/` |

หน้าแยกจะส่งต่อไปยัง `?p=1` / `?p=2` / `?p=3` โดยอัตโนมัติ และเปิดภาพกล้องหลังผู้ชมอนุญาตสิทธิ์ทันที
จะใช้ `?s=tulip` / `?s=butterfly` / `?s=mushroom` โดยตรงก็ได้ผลเหมือนกัน (อิงตาม `id` ใน `SPECIMENS`)

ผู้ชมยังต้องแตะ “เปิด AR” หนึ่งครั้งหลังกล้องเปิด เพราะ iOS ต้องขอสิทธิ์เซ็นเซอร์การหมุน และ Android ต้องเริ่ม immersive AR ภายใน user gesture เท่านั้น

## วางบนพื้น หรือ วางบนโต๊ะ

หน้าแรกให้เลือกพื้นผิวก่อนเลือกตัวอย่าง เพราะขนาดต่างกันมาก

| | เห็ด | กระเจียว | ผีเสื้อ |
|---|---|---|---|
| พื้นห้อง | 58 ซม. | 72 ซม. | กระจาย 2.4 ม. ลอยสูง 1.3 ม. |
| บนโต๊ะ | 16 ซม. | 20 ซม. | กระจาย 67 ซม. ลอยสูง 36 ซม. |

ปรับได้ที่ `const SURFACES`

- `scale` ตัวคูณขนาด โหมดโต๊ะตั้งไว้ 0.28
- `drop` ระยะจากกล้องลงไปถึงพื้นผิว **ใช้เฉพาะ iOS** ตั้งไว้ 0.55 ม. คิดจากคนยืนถือมือถือมองโต๊ะสูง 75 ซม.
  ถ้าทดสอบแล้ววัตถุจมลงไปในโต๊ะให้ลดค่านี้ ถ้าลอยเหนือโต๊ะให้เพิ่ม
  บน Android ไม่ต้องยุ่ง เพราะ WebXR อ่านพื้นผิวจริงเอง

## เรื่อง marker

ระบบนี้ **ไม่ใช้ marker** เล็งกล้องไปที่พื้นผิวแล้วกดวางได้เลย

ถ้าจะเพิ่มโหมด image target ทีหลัง ไฟล์ที่ต้องใช้คือ **ภาพแบนที่พิมพ์ออกมาจริง**
เช่นไฟล์อาร์ตเวิร์กป้ายนิทรรศการ ไม่ใช่ภาพถ่ายฉากหรือภาพถ่ายพื้นผิว
ภาพถ่ายฉาก 3 มิติจะจับติดเฉพาะมุมที่ถ่ายไว้ และพังทันทีที่ของในฉากถูกขยับ

ภาพที่เหมาะเป็น target ควรมี ลวดลายไม่ซ้ำกันเอง คอนทราสต์สูง ไม่สมมาตร และไม่ใช่พื้นสีเรียบ

## รูปแบบวิดีโอ

คลิปเป็น **alpha-packed**: ครึ่งบนของเฟรมคือภาพสี ครึ่งล่างคือ alpha matte
เพราะ mp4 ปกติไม่มี alpha channel วิธีนี้เล่นได้ทั้ง iOS และ Android ด้วย H.264 ธรรมดา

สร้างคลิปใหม่จากบลูสกรีนด้วยคำสั่งนี้ (แก้ค่า `KEY` `SIM` `BLEND` ตามคลิป)

```bash
ffmpeg -i input.mp4 -filter_complex "
  [0:v]scale=960:540,format=rgba,colorkey=0x097BF7:0.22:0.14,
       despill=type=blue:mix=1.0:expand=0.6[k];
  [k]split[k1][k2];
  color=c=black:s=960x540:r=24[bg];
  [bg][k1]overlay=shortest=1,format=gbrp[r];
  [k2]alphaextract,format=gbrp[a];
  [r][a]vstack=inputs=2,format=yuv420p[o]" \
  -map "[o]" -c:v libx264 -crf 20 -preset slow -profile:v main -level 4.0 \
  -g 48 -pix_fmt yuv420p -movflags +faststart -an output.mp4
```

ค่าที่ใช้กับสามคลิปนี้

| คลิป | key | similarity | blend |
|---|---|---|---|
| เห็ด | `0x2D67C1` | 0.18 | 0.09 |
| ผีเสื้อ | `0x3488E5` | 0.22 | 0.14 |
| กระเจียว | `0x097BF7` | 0.22 | 0.14 |

คลิปผีเสื้อมีแถบดำตกค้างขอบบนล่าง 12 px ต้องใส่ `crop=1920:1056:0:12,pad=1920:1080:0:12:0x3488E5,` นำหน้า `scale`

เห็ดต้องใช้ similarity ต่ำกว่าเพื่อน เพราะพื้นหลังฟ้าอมเทาใกล้เคียงสีดินและเงา
ถ้าดันเกิน 0.19 คีย์จะเริ่มกินหมวกเห็ด

## สองโหมดการทำงาน

แอปเลือกโหมดเองอัตโนมัติ

- **Android Chrome** ใช้ WebXR hit-test จับพื้นจริง เดินรอบได้แม่นยำ
- **iOS Safari** ไม่รองรับ WebXR จึงใช้กล้อง + เซ็นเซอร์การหมุนแทน แตะวางแล้ววัตถุจะตรึงอยู่กับที่เวลาหันเครื่อง แต่ไม่ได้อ่านพื้นจริง จึงกะระยะจากความสูงกล้องโดยประมาณ (ตัวแปร `EYE` = 1.45 ม.)

ถ้าอยากได้ SLAM จริงบน iOS ต้องใช้บริการเสียเงินอย่าง 8th Wall / Niantic Studio / Zappar

## ต้องทดสอบก่อนเปิดจริง

โค้ดนี้ตรวจ syntax และตรรกะเชเดอร์แล้ว แต่ยังไม่ได้ลองบนเครื่องจริง ควรเช็ค

- [ ] iPhone Safari — ขออนุญาตเซ็นเซอร์การหมุนผ่านไหม วัตถุตรึงอยู่กับที่ไหม
- [ ] Android Chrome — WebXR เข้าโหมดได้ไหม เรติเคิลเกาะพื้นไหม
- [ ] เวลาโหลดคลิปแรกบน 4G หน้างาน
- [ ] แสงในห้องจัดแสดง ถ้ามืดมากกล้องจะ noise และ WebXR จะจับพื้นไม่ติด
- [ ] ขนาดวัตถุเทียบของจริง ปรับ `width` ให้สมจริง

## ถ้าไม่เห็นวัตถุ

1. เปิด `/diag.html` บนเครื่องที่มีปัญหา แล้วกด “เริ่มตรวจ” จะไล่ทดสอบทีละขั้นและบอกว่าล้มตรงไหน
2. เปิดแอปพร้อม `?debug=1` ต่อท้าย URL จะมี HUD มุมล่างซ้ายบอกสถานะวิดีโอและตำแหน่งวัตถุแบบเรียลไทม์

สาเหตุที่เจอบ่อย

| อาการใน diag | แปลว่า |
|---|---|
| ไฟล์ video/*.mp4 หา 404 ไม่เจอ | อัปไม่ครบ หรือ path ผิด |
| สั่งเล่นวิดีโอ → NotAllowedError | เปิดจากลิงก์ในแอป LINE/Facebook ให้กด “เปิดใน Safari/Chrome” ก่อน |
| สิทธิ์เซ็นเซอร์ → denied | iOS ต้องกดอนุญาตตอนที่ถาม ถ้าเผลอกดปฏิเสธไปแล้ว ต้องล้างค่าที่ ตั้งค่า → Safari → ขั้นสูง → ข้อมูลเว็บไซต์ |
| เซ็นเซอร์ไม่ส่งค่าจริง | เปิดที่ ตั้งค่า → Safari → การเคลื่อนไหวและทิศทาง |
| ทุกข้อผ่านแต่ยังไม่เห็น | ปัญหาอยู่ที่ตำแหน่ง/ขนาด ใช้ `?debug=1` ดูค่า obj แล้วส่งมา |
